# 盘面信号监控 — 常见踩坑记录

## 1. 腾讯财经指数代码前缀问题

**问题**：上证指数显示为 11.06（平安银行价格），沪深300 显示为 0.00

**原因**：腾讯财经 API 的指数代码需要特殊市场前缀：
- 上证指数: `sh000001`（不是 `sz000001`）
- 沪深300: `sh000300`（不是 `sz000300`）
- 中证500: `sh000905`（不是 `sz000905`）
- 创业板指: `sz399006`
- 国证2000: `sz399303`

**解决**：在 `tencent_quote()` 函数中添加 `INDEX_MAP` 字典，对指数代码做特殊映射。

## 2. 腾讯 API 返回 key 格式问题

**问题**：解析出的股票代码包含 `=` 号（如 `000001=`）

**原因**：腾讯 API 返回的格式是 `sh0000001=...`，split 后 key 会包含 `=`。

**解决**：解析时去掉 `=` 号：
```python
code = key.replace("=", "")
if code.startswith(("sh", "sz", "bj")):
    code = code[2:]
```

## 3. 非交易时间数据返回 0

**问题**：涨跌家数、涨停数在周末/盘后返回 0

**原因**：东财 push2 的全市场统计数据只在交易时间有实时数据

**解决**：这是正常现象，报告中会显示 "0家"。交易时间运行时数据正常。

## 4. 北向资金数据延迟

**问题**：北向资金数据有时显示负值或异常值

**原因**：同花顺 hsgtApi 返回的是分钟级累计数据，开盘初期数据不稳定

**解决**：盘前（9:15）运行时北向资金可能还未开始流入，显示为 0 或负值是正常的。

## 5. 东财 API 限流

**问题**：频繁调用东财 API 导致 403/429

**原因**：东财有风控：每秒>5 次、单 IP 并发≥10、1 分钟≥200 次

**解决**：已内置 `em_get()` 统一请求入口，自动限流（间隔≥1 秒 + 随机抖动）。

## 6. 融资余额数据精度

**问题**：融资余额显示为 56 亿（明显偏低）

**原因**：东财 datacenter 返回的是个股明细，需要按市场汇总

**说明**：当前实现是查询全市场汇总数据，如果返回异常可能是 API 过滤条件问题。

## 7. VPN/代理导致腾讯 API SSL 断连

**问题**：`tencent_quote()` 调用抛出 `SSLEOFError: EOF occurred in violation of protocol` 或 `UNEXPECTED_EOF_WHILE_READING`

**原因**：VPN/代理软件（Surge/Clash 等）将 `qt.gtimg.cn` DNS 劫持到虚拟 IP（`198.18.x.x`），导致 SSL 握手间歇性失败。系统 Python 3.9 + LibreSSL 2.8.3 更容易触发。

**解决**：
1. 使用 Python 3.11+（OpenSSL 3.x）
2. `tencent_quote()` 已内置 3 次重试 + 2s 间隔
3. VPN 分流规则将 `qt.gtimg.cn` 加入直连列表可根治

## 8. 开盘初期涨跌数据为 0

**问题**：09:30–09:35 运行时涨跌家数、涨停/跌停数均为 0

**原因**：东财 push2 全市场统计数据在开盘后 3-5 分钟才刷新

**解决**：正常现象，不影响其他信号。首次监控建议 09:35 后运行。
